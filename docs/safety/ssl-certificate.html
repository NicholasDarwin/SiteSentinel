SSL Certificate Status

Critical Severity

What This Check Measures

This check verifies whether the website uses HTTPS (HTTP Secure) encryption, which indicates that an SSL/TLS certificate is active and the connection between the user and server is encrypted.

Why It Matters

  • Data Privacy ‚Äî HTTPS encrypts all data transmitted between the browser and server
  • Authentication ‚Äî SSL certificates verify the website's identity
  • Integrity ‚Äî Prevents man-in-the-middle attacks and data tampering
  • SEO Impact ‚Äî Google ranks HTTPS sites higher in search results
  • Browser Trust ‚Äî Modern browsers show "Not Secure" warnings for HTTP sites

Status Values

Status Condition Meaning
‚úÖ pass URL starts with https:// HTTPS is enabled. Data is encrypted in transit.
‚ùå fail URL starts with http:// No encryption. All data sent in plain text. Critical security risk.

Severity: Critical

This check has critical severity because:

  • HTTP connections expose all traffic to eavesdropping
  • Passwords, credit cards, and personal data can be intercepted
  • Users may unknowingly submit sensitive information over insecure connections
  • Modern browsers actively warn users about non-HTTPS sites

Impact on Category Score

As a critical severity check:

  • A pass contributes positively to the Safety & Threats score
  • A fail significantly reduces the category score
  • The calculateCategoryScore() function weights critical checks heavily
